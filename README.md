# Federal Court Complexity vs Efficiency

University project: modeling the relationship between **case complexity** and **operational efficiency (LOS)** using the SCALES federal court event log (Northern District of Illinois).

**Research question:** Is there a link between case complexity and procedural efficiency (LOS), and does it differ between civil and criminal cases and across judges?

---

## Prerequisites

1. Place `Event Log.csv` in the project root (~2.7 GB, not in git). `Event Log_model.csv` (~2.5 GB, generated by the pipeline) is also kept local only.
2. Python 3.10+ recommended.
3. On **macOS**, XGBoost requires OpenMP:
   ```bash
   brew install libomp
   ```

---

## Setup (one time)

```bash
cd /path/to/Project
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

## Run the full pipeline

### Recommended: civil + criminal + pooled models

```bash
bash scripts/run_pipeline.sh --by-case-type
```

This runs all 6 steps: event cleaning → case features → aggregations → RF/DT → XGBoost+SHAP → causal regression, for **pooled**, **civil (cv)**, and **criminal (cr)** cases.

### Pooled models only (faster)

```bash
bash scripts/run_pipeline.sh
```

### Fast dev run (500k event rows)

```bash
bash scripts/run_pipeline.sh --sample --by-case-type
```

### Re-run models only (data already built)

```bash
bash scripts/run_pipeline.sh --skip-clean --skip-case --by-case-type
```

### Pipeline options

| Flag | Effect |
|------|--------|
| `--by-case-type` | Build `by_*_cv.parquet` / `by_*_cr.parquet` and run models separately for civil and criminal |
| `--sample` | Use 500,000 event rows (quick test) |
| `--skip-clean` | Reuse existing `Event Log_model.csv` |
| `--skip-case` | Reuse existing `data/case_features.parquet` |
| `--skip-models` | Build data only; skip ML / SHAP / regression |
| `-h`, `--help` | Show help |

`scripts/run_extended_pipeline.sh` is an alias for `run_pipeline.sh`.

## Pipeline steps (manual)

If you prefer to run each step yourself:

```bash
source .venv/bin/activate

# 1. Clean raw event log (closed cases only)
python scripts/build_features.py

# 2. Case-level complexity + LOS (one row per ucid)
python src/build_case_features.py

# 3. Aggregations: by case, judge, city (+ cv/cr with --by-case-type)
python src/build_aggregations.py --by-case-type

# 4. Random Forest + Decision Tree feature importance
python scripts/run_ml_importance.py --case-type all
python scripts/run_ml_importance.py --case-type cv
python scripts/run_ml_importance.py --case-type cr

# 5. XGBoost + SHAP
python scripts/run_xgb_shap.py --case-type all
python scripts/run_xgb_shap.py --case-type cv
python scripts/run_xgb_shap.py --case-type cr

# 6. Causal-style OLS regression (complexity → log LOS)
python scripts/run_causal_regression.py --case-type all
python scripts/run_causal_regression.py --case-type cv
python scripts/run_causal_regression.py --case-type cr
```

## Pipeline flow

```
Event Log.csv
  └─ scripts/build_features.py          → Event Log_model.csv (events, closed only)
       └─ src/build_case_features.py    → data/case_features.parquet (cases)
            └─ src/build_aggregations.py → data/aggregations/by_*.parquet

                 ├─ scripts/run_xgb_shap.py           (XGBoost + SHAP)

```

## Key definitions

| Term | Meaning |
|------|---------|
| **LOS** | Length of stay — days from first to last `date_filed` per closed case |
| **Complexity** | `n_events`, `n_activity_types`, `n_motions`, `activity_entropy`, attribute sums |
| **cv / cr** | Civil / criminal case type |
| **is_mdl** | Multi-District Litigation flag (rare; long-running cases) |
| **M1 / M2 / M3** | OLS models with increasing controls (complexity only → + case controls → + judge fixed effects) |

Analysis uses **closed cases only**; open cases are excluded at event cleaning (censored for LOS).

## Troubleshooting

**XGBoost fails on macOS (`libomp.dylib not found`):**
```bash
brew install libomp
```

**Out of memory:** use `--sample` or run steps individually with `--skip-clean` / `--skip-case`.

**Re-build cleaned CSV from scratch:** delete `Event Log_model.csv` and run without `--skip-clean`.

---

## MDL sensitivity & report tables

MDL cases are rare (~4%) but have **much longer LOS** (median 894 vs 257 days). Re-run without them:

```bash
python scripts/run_mdl_sensitivity.py
```

Or rebuild tables only (after `_no_mdl` JSON files exist):

```bash
python scripts/build_results_tables.py
```

Add `--exclude-mdl` to any model script:

```bash
python scripts/run_xgb_shap.py --case-type cv --exclude-mdl
```

**Key finding:** pooled predictive R² drops when MDL is removed (models leaned on `is_mdl`); **civil** metrics change modestly; **criminal** is unchanged (0 MDL cases in sample). Causal OLS R² is **stable or slightly higher** without MDL.

```bash
python scripts/run_survival_analysis.py --case-type cv
python scripts/run_survival_analysis.py --case-type cr
```
